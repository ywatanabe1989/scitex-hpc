The full scitex-hpc command-line surface is rendered directly from the live click group via sphinx-click.

scitex-hpc

scitex-hpc (v0.8.3) — scitex-hpc - Generic SLURM dispatch + persistent reservations.

Config is loaded with the SciTeX precedence chain:
config.yaml -> $SCITEX_HPC_CONFIG -> ~/.scitex/hpc/config.yaml -> defaults

Usage

scitex-hpc [OPTIONS] COMMAND [ARGS]...

Options

-V, --version

Show the version and exit.

-h, --help

Show this message and exit.

--help-recursive

Show help for all commands recursively.

--json

Emit structured JSON output (propagates to subcommands that honour it).

install-shell-completion

Wire up <TAB> completion in the user’s shell rc.

Examples:
scitex-hpc install-shell-completion # → ~/.bashrc
scitex-hpc install-shell-completion –shell zsh # → ~/.zshrc
scitex-hpc install-shell-completion –dry-run # preview only
Activate in the current shell after install:
source ~/.bashrc

Usage

scitex-hpc install-shell-completion [OPTIONS]

Options

--shell <shell>

Target shell. Default: bash.

Options:

bash | zsh | fish

--dry-run

Print the eval line and target rc file without writing.

-y, --yes

Skip confirmation prompt.

reservations

Persistent SLURM allocations (book once, exec many).

Usage

scitex-hpc reservations [OPTIONS] COMMAND [ARGS]...

attach

Open an interactive shell on the reservation’s compute node.

Example:
$ scitex-hpc reservations attach dev-pool
$ scitex-hpc reservations attach dev-pool –shell zsh

Usage

scitex-hpc reservations attach [OPTIONS] NAME

Options

--host <host>
--shell <shell>

Arguments

NAME

Required argument

book

Submit a hold-job and wait for SLURM allocation.

Naming convention (use this for NAME — encodes the shape so any
operator can read it without re-discovering the allocation):
CPU-only : cpu-<cores>-cores-<ram_gb>-ram
e.g. cpu-64-cores-256-ram
GPU : gpu-<cores>-cores-<ram_gb>-ram-<vram_gb>-vram
e.g. gpu-16-cores-128-ram-80-vram (1× H100, 80 GB)
gpu-32-cores-256-ram-40-vram (1× A100-40, etc.)
Suffix -NxTYPE if you book multiple GPUs (e.g. …-80-vram-4xh100).
Persistent reservations should also embed the cluster prefix
(spartan-cpu-64-cores-256-ram) so multi-cluster operators don’t
collide on the same lease id.
Example:
$ scitex-hpc reservations book spartan-cpu-64-cores-256-ram
–host spartan –partition cascade
–cpus 64 –mem 256G –time 7-0
–account punim2354 –persistent
$ scitex-hpc reservations book spartan-gpu-16-cores-128-ram-80-vram
–host spartan –partition gpu-h100
–cpus 16 –mem 128G –gpus h100:1 –time 7-0
–account punim2354 –qos feit –persistent

Usage

scitex-hpc reservations book [OPTIONS] NAME

Options

--host <host>

SSH host to submit sbatch from (e.g. spartan). Optional — falls back to $SCITEX_HPC_HOST or ~/.scitex/hpc/config.yaml’s host: if unset, so an operator with a single cluster doesn’t need to pass this every time.

--partition <partition>
--cpus <cpus>
--time <time_>

walltime, e.g. 7-0 or 01:00:00

--mem <mem>
--nodelist <NODE>

Pin the allocation to a specific node (e.g. spartan-bm198). SLURM will wait for the named node to free up rather than scheduling elsewhere.

--account <account>

SLURM account / project to bill (e.g. punim2354).

--qos <qos>

SLURM QOS tier (e.g. publiccpu).

--gpus <SPEC>

Request GPUs. Pass-through to SLURM’s –gpus=<SPEC>. Examples: ‘1’ (any 1 GPU), ‘a100:2’ (two A100s), ‘h100:4’. Omit for CPU-only allocations.

--persistent

walltime auto-resubmit via SIGUSR1 (Phase 2).

--hold-body <hold_body>

Custom sbatch script body (default: tail -f /dev/null).

--tmux-server <SOCKET>

Bootstrap a long-lived tmux server with this socket name as the job’s PID 1. Required for scitex-agent-container’s slurm-tenant runtime. Example: –tmux-server sac

--poll-timeout <poll_timeout>
--poll-interval <poll_interval>
--dry-run

Print plan without sbatch’ing.

-y, --yes

Skip confirmation prompt.

--json

Emit JSON output.

NAME

Required argument

Options

--json

NAME

Required argument

install

Install this package’s skills into a target directory.

Example:
$ scitex-hpc skills install
$ scitex-hpc skills install –claude-symlink
$ scitex-hpc skills install –no-link –dest /tmp/scitex-hpc-skills

Usage

scitex-hpc skills install [OPTIONS]

Options

--dest <dest>

Destination dir (default: ~/.scitex/dev/skills/scitex-hpc/).

--no-link

Copy files instead of symlinking. Default is symlink.

--claude-symlink

Also expose at ~/.claude/skills/scitex/ for Claude Code consumers.

--dry-run

Preview without copying/linking.

-y, --yes

Skip confirmation prompt.
